# [PRD] AI 기반 즉흥 활동 스케줄링 앱: SpotPlan

## 1. 제품 개요
* **제품명:** SpotPlan (가칭)
* **핵심 기능:** 사진 분석(Vision AI)을 통한 현재 위치 파악 및 가용 시간 맞춤형 스케줄링 제공.
* **주요 가치:** 낯선 환경에서의 즉흥적인 활동 계획 수립 지원 및 이동 편의성 제공.

## 2. 사용자 시나리오 (User Story)
1.  **위치 파악:** 사용자는 낯선 장소(역, 정류장, 랜드마크 등)를 촬영하거나 사진을 업로드하여 현재 위치를 식별합니다.
2.  **시간 설정:** 사용자는 해당 지역에서 머무를 시간(1~8시간)을 입력합니다.
3.  **계획 수립:** AI는 위치와 시간을 바탕으로 최적의 활동 동선(Todo-list)을 제안합니다.
4.  **활동 수행:** 사용자는 체크리스트를 활용해 활동을 관리하고, 아코디언 메뉴를 통해 상세 가이드를 확인합니다.
5.  **길 안내:** 리스트 아이템 클릭 시 구글 맵으로 즉시 연결되어 목적지까지 네비게이션을 지원받습니다.

## 3. 기능적 요구사항

### 3.1 사용자 입력 (Input)
* **이미지 데이터:**
    * 인앱 카메라 실시간 촬영 및 업로드.
    * 디바이스 갤러리 내 저장된 사진 선택.
* **활동 설정:** 1시간에서 8시간 사이의 가용 시간 입력.
* **언어 설정:** 한국어(기본) 및 영어 모드 지원.
* **길찾기** : 해당 장소 클릭하면 현재 위치부터 해당 장소까지 구글맵으로 네비게이션 적용하여 구글맵 실행.

### 3.2 AI 및 백엔드 로직
* **Vision 분석 (Gemini):** 이미지 내 랜드마크, 간판, 역명 등을 분석하여 좌표 및 장소명 추출.
* **현재 사용자 위치 확인:** 이미지 분석으로 인한 위치와 내 현재 위치(GPS)를 이용해서 현재 위치 보정.
* **데이터 연동 (Google Search & Maps):**
    * 식별된 위치 주변의 실시간 정보(맛집, 카페, 명소 등) 수집.
    * 실시간 영업 정보, 날씨, 트렌드 데이터 반영.
* **스케줄링 엔진:**
    * **필터링:** 영업 종료, 폐업, 불분명하거나 위험한 장소 자동 제외.
    * **최적화:** 입력된 시간 내 이동 시간을 고려한 밀도 있는 일정 생성.

### 3.3 출력 및 UX (Output)
* **상단 정보:** 분석된 위치의 명칭과 사용자가 업로드한 이미지 반드시 표시.
* **스케줄 리스트:**
    * 시간순으로 정렬된 활동 Todo-list (체크박스 포함).
    * **아코디언 UI:** 각 항목 클릭 시 해당 장소에서의 구체적인 활동 제안(Plan) 노출.
	* URL이 외부로 나오면 안됨.
* **외부 앱 연동:** 리스트 터치 시 현재 GPS 위치에서 목적지까지 구글 맵 네비게이션 실행.
* **텍스트 출력:** 리스트 아이템 및 각 영역에 텍스트가 잘리지 않게끔 처리.

## 4. 제약 및 준수 사항
* **데이터 근거:** 모든 장소 추천 및 스케줄링은 실시간 검색 결과를 기반으로 해야 함.
* **안전성:** 폐업한 장소나 안전이 보장되지 않는 장소는 추천 목록에서 원천 배제.
* **정확성:** 사용자의 가용 시간을 초과하지 않는 합리적인 동선 설계, 반드시 현재 시간을 기준하여 처리할것.
* 응답속도를 최적화 하기위해 적절한 모델 설정

## 5. 기술 스택 가이드 (AI Studio 설정용)
* **Model:** Gemini 2.5 Flash / Pro.
* **Tools:** Google Search Grounding 활용
* **Prompting:** * 이미지 속 텍스트와 주변 지형물을 기반으로 위경도를 추론할 것.
    * 사용자의 입력 시간에 따라 활동의 개수를 유동적으로 조절할 것.
    * 결과값에 구글 맵 네비게이션 연결을 위한 `place_id` 또는 좌표를 포함할 것.
